Exercise diary mobile application

It as a smart notebook for gym exercices where you can add, remove and procces your trainings. The app does not use hosting servers and all trainings data is stored on the device, therefore data privacy is guaranteed. The idea of the app is based on 4 different approaches to exercising: usual training, weight progression training, curcuit training and cardio. All info about the implementation of that idea you may find below.

Home screen

There are six buttons each of which will transfer you to appropriate screen. All screens of the app was built using kivy framework that is convenient for widgets accommodation. In generall, the app contains text input widgets, buttons, images, spinners, graphs and accordion widgets that are the container of buttons that transfer you to one of the training recording screens

Plans screen

Each time Plans screen is Initialized, a new row in SQL database is created with ID returned by a Python function that compiles current date, training number and training type by default. This ID is used in all functions that request the data of the trainings. Therefore ID is a pillar of the application

Workout screen

Shows all trainings information from database for the nearest day to today by default. The key distinction of the workout screen is that it initializes the widgets after user enters the screen and not before. All info shown on that screen depends on the button that called it. It might be called from either home screen or training history screen.

Training history screen

Shows all training recordings in a chosen interval and filters them by the type. User may choose interval to filter his or her trainings by embedded calendar implemented by Kivy Material Design. All widgets that appear after applying filters are buttons leading to workout screen.

Results screen

Uses matplotlib library and FigureCanvasKivyAgg backend to show the graph of generated by Python function values of stamina and power. The overall efficiency might be estimated by a special coefficient that combines both stamina and power values.

Four types of training

Four different ways to record user's training, four diffrent approaches to compile widgets on the screen. Each spinner widget bears the utility to dynamically change records in the database. On top, some widgets affect other widgets. For example, if the user chooses "chest" as a muscle group, exercise widget will offer to chose only "chest exercises" in a dropdown menu. If the user chooses "Bench Press" exercise, the adjacent weight widget will offer to choose weight in range between 10 (the lightiest weight for bench press) and 250 killograms with intervals of 2,5 kg. In adition, there are two databases: temporary where data is recorded as the user interacts with widgets and permanent where data is transferred from temporary database after button "Save to history" is clicked.
